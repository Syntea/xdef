X-definice

Úvod

X-definice je technologie určená k popisu validace, zpracování a konstrukce dat ve formátu XML (a také JSON, YAML a podobné).

X-definition je technologie navržená společností Syntea software group a.s. pro profesionální popis, validaci, zpracování a tvorbu struktury dokumentů XML, JSON, YAML, INI a CSV. Umožňuje uživateli nejen definovat strukturu XML-dokumentů, ale také konkrétně popsat jejich zpracování a konstrukci. X-definition také umožňuje definovat a používat kód v externích uživatelských metodách.

Tento projekt je implementován pro platformu Java 1.8+ (doplňující poznámka: až do verze 41.0.4 pro platformu Java 1.6+).

Názorné základní příklady

Příklad 1: Základní konstrukce
Přčdpokládejme následující XML data: 
<Employee
  FirstName = "Andrew"
  LastName  = "Aardvark"
  EnterDate = "1996-03-12"
  Salary    = "21700"
>
  <Address
    Street = "Broadway"
    Number = "255"
    Town   = "Beverly Hills"
    State  = "CA"
    Zip    = "90210"
  />
  <Competence>electrician</Competence>
  <Competence>carpenter</Competence>
</Employee>
Toto je kompletní X-definice popisující XML data vlevo: 
<xd:def
  xmlns:xd = "http://www.xdef.org/xdef/4.2"
  xd:root  = "Employee"
>
  <Employee
    FirstName = "required string()"
    LastName  = "required string()"
    EnterDate = "required date()"
    Salary    = "optional decimal()"
  >
    <Address
      Street = "required string()"
      Number = "required int()"
      Town   = "required string()"
      State  = "required string()"
      Zip    = "required int()"
    />
    <Competence xd:script = "occurs 1..5">
      required string()
    </Competence>
  </Employee>
</xd:def>

Anotace

X-definice je určena pro popis a zpracování dat ve formátu XML a také JSON, YAML a podobných formátech.

X-definice je open-source nástroj, který popisuje strukturu a vlastnosti datových hodnot v dokumentu XML. X-definice navíc umožňuje popsat zpracování a konstrukci objektů XML. X-definice tak může nahradit stávající technologie běžně používané pro validaci XML – konkrétně Data Type Definition (DTD), XML Schema (XSD), Schematron a Extensible Stylesheet Language Transformations (XSLT).

X-definice umožňuje kombinovat validaci dokumentů XML se zpracováním dat (popisem akcí přiřazených každé události při zpracování objektů XML). Ve srovnání s technologiemi založenými na DTD a schématu XML je výhodou X-definice (nejen) vyšší čitelnost a snadnější údržba. X-definice byla navržena tak, aby zvládala datové sady XML prakticky neomezené velikosti přesahující velikost pracovní paměti.

Důležitou vlastností X-definice je maximální respektování struktury popisovaných dat. Forma X-definice je XML dokument se strukturou podobnou popisovaným XML datům. To umožňuje rychlý a intuitivní popis XML dat a jejich zpracování.

Vlastnosti XML položek (a události, které mohou během procesu nastat) jsou popsány jazykem X-script. Ve většině případů stačí nahradit hodnoty popsané v datovém modelu XML jazykem X-script v X-definici. Do X-scriptu můžete také postupně přidávat požadované akce zpracování dat. Technologie X-definice také umožňuje generovat zdrojový kód pro třídy reprezentující XML prvky popsané X-definicí v Javě. Taková třída se nazývá X-komponenta. Instance XML dat lze použít ve formě X-komponent (podobně jako technologie JAXB).

Počínaje verzí X-definice 4.2 je možné kromě XML ověřovat a zpracovávat data ve formátech JSON a YAML, Properties, Windows INI a CSV (hodnoty oddělené čárkami). Modely datových struktur JSON jsou popsány elementy xd:json nebo xd:ini.

X-definice také umožňuje jazykovou lokalizaci názvů tagů popisovaných XML dat. Různé jazykové verze lze popsat elementy v objektu xd:lexicon.
