X-definición

Introducción

X-definición es una Tecnología diseñada para describir la validación, el procesamiento y la construcción de datos en formato XML (también JSON, YAML y similares).

X-definición es una tecnología desarrollada por Syntea Software Group a.s. para la descripción, validación, procesamiento y creación profesional de estructuras de documentos XML, JSON, YAML, INI y CSV. Permite al usuario no solo definir la estructura de los documentos XML, sino también describir específicamente su procesamiento y construcción. X-definición también permite definir y utilizar código en métodos externos de usuario.

Este proyecto está implementado para la plataforma Java 1.8+ (nota adicional: hasta la versión 41.0.4 para Java 1.6+).

Ejemplos básicos ilustrativos

Ejemplo 1: Estructura básica
Supongamos los siguientes datos XML: 
<Empleado
  Nombre   = "Andrew"
  Apellido = "Aardvark"
  Ingreso  = "1996-03-12"
  Salario  = "21700"
>
  <Dirección
    Calle  = "Broadway"
    Número = "255"
    Ciudad = "Beverly Hills"
    Estado = "CA"
    CódigoPostal = "90210"
  />
  <Competencia>electricista</Competencia>
  <Competencia>carpintero</Competencia>
<Empleado>
Esta es la X-definición completa que describe los datos XML de la izquierda: 
<xd:def
  xmlns:xd = "http://www.xdef.org/xdef/4.2"
  xd:root  = "Empleado"
>
  <Empleado
    Nombre   = "required string()"
    Apellido = "required string()"
    Ingreso  = "required date()"
    Salarios = "optional decimal()"
  >
    <Dirección
      Calle  = "required string()"
      Número = "required int()"
      Ciudad = "required string()"
      Estado = "required string()"
      CódigoPostal = "required int()"
    />
    <Competencia xd:script = "occurs 1..5">
      required string()
    </Competencia>
  </Empleado>
</xd:def>

X-definición que describe los datos XML de la izquierda: 
<xd:def
  xmlns:xd = "http://www.xdef.org/xdef/4.2"
  xd:root  = "Familia"
>
  <Familia>
    <Padre      xd:script = "occurs 0..1; ref Persona" />
    <Madre      xd:script = "occurs 1..1; ref Persona" />
    <Hijo       xd:script = "occurs 0..*; ref Persona" />
    <Hija       xd:script = "occurs 0..*; ref Persona" />
    <Residencia xd:script = "occurs 1;    ref Dirección" />
  </Familia>

  <Persona
    Nombre     = "string()"
    Apellido   = "string()"
    IDPersonal = "long()"
    Salario    = "opcional int()"
  />
  <Dirección
    Calle  = "string()"
    Número = "int()"
    Ciudad = "string()"
    CódigoPostal = "int()"
  />
<xd:def>

Anotación

X-definición está diseñado para la descripción y el procesamiento de datos en formato XML, así como en JSON, YAML y formatos similares.

X-definición es una herramienta de código abierto que describe la estructura y las propiedades de los valores de datos en un documento XML. Además, X-definición permite describir el procesamiento y la construcción de objetos XML. Por lo tanto, X-definición puede reemplazar las tecnologías existentes comúnmente utilizadas para la validación XML, como la Definición de Tipos de Datos (Data Type Definition — DTD), el XML-Esquema (XML-Schema — XSD), Schematron y las Transformaciones del Lenguaje de Hojas de Estilo Extensibles (Extensible Stylesheet Language Transformations — XSLT).

X-definición permite combinar la validación de documentos XML con el procesamiento de datos (descripción de las acciones asignadas a cada evento al procesar objetos XML). En comparación con las tecnologías basadas en DTD y esquemas XML, la ventaja de X-definición radica (no solo) en una mayor legibilidad y un mantenimiento más sencillo. X-definición se diseñó para manejar conjuntos de datos XML de tamaño prácticamente ilimitado, que superan la capacidad de la memoria RAM.

Una característica importante de X-definición es el máximo respeto por la estructura de los datos descritos. El formato de una X-definición es un documento XML con una estructura similar a la de los datos XML descritos. Esto permite una descripción rápida e intuitiva de los datos XML y su procesamiento. Las propiedades de los elementos XML (y los eventos que pueden ocurrir durante el proceso) se describen en el lenguaje X-script.

En la mayoría de los casos, basta con reemplazar los valores descritos en el modelo de datos XML con el lenguaje X-script en la X-definición. También se pueden añadir gradualmente las acciones de procesamiento de datos necesarias al X-script. La tecnología X-definición también permite generar código fuente para clases que representan elementos XML descritos por la X-definición en Java. Dicha clase se denomina componente X. Las instancias de datos XML se pueden utilizar en forma de componentes X (similares a la tecnología JAXB).

A partir de X-definición 4.2, es posible validar y procesar datos en formatos JSON y YAML, propiedades, archivos INI de Windows y CSV (valores separados por comas), además de XML. Los modelos de estructura de datos JSON se describen mediante los elementos xd:json o xd:ini.

X-definición también permite la localización de idiomas para los nombres de etiquetas descritos en los datos XML. Las diferentes versiones de idioma se pueden describir mediante elementos del objeto xd:lexicon.
